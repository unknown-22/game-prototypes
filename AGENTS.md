# instructions

**日本語で返答してください**

ゲームアイデアの生成とプロトタイプ作成を行うモノレポ。
prototypes/ 以下に 001_<プロトタイプ名>/main.py という構成でプロトタイプを配置してください。

## 主要技術スタック・コーディング規約

- 言語: Python 3.12
- フレームワーク: pyxel

**プロジェクトルール**

- python 3.12のモダンな機能を活用する
- 型ヒントを積極的に使用する
- ruff(lint)とty(type checker)を活用してコード品質を保つ
- 適切な単位でモジュール化・クラス化を行う
- ロジック部分はクラス化・関数化してテスト可能にする
- linterや型チェックで関数・引数ミスが出たら inspect モジュールで関数シグネチャを確認するワンライナーを都度作成し、調査と修正を繰り返す
- 作成後に docs/prototypes.json を更新する

## ゲームプロトタイプ設計指針

プロトタイプでは完成度や物量よりも、「遊びの核」が短時間で伝わることを優先する。
プレイヤーが判断し、操作し、結果を受け取り、次はもっと上手くやろうと思えるループを必ず含める。

**ゲームとして面白くするための観点**

- プレイヤーの選択に意味を持たせる
  - 「リスクを取るか安全に進めるか」「どの順番で処理するか」「今リソースを使うか温存するか」のような判断を入れる
  - 良い選択は明確に得をし、悪い選択は納得できる損につながるようにする
- 短いプレイループを作る
  - 見る → 判断する → 操作する → 結果が返る → 次の判断へ、という流れを数秒から十数秒で回す
  - プロトタイプ中に同じ面白さが何度も発生するようにする
- 失敗理由をわかりやすくする
  - 敵の予兆、危険範囲、残り時間、HP、リソース不足などを画面上で伝える
  - 理不尽な失敗ではなく、次はどう改善できるかが見える失敗にする
- 成功を気持ちよく見せる
  - スコア加算、敵撃破、連鎖、回避成功、強化獲得などに即時フィードバックを入れる
  - Pyxelでは色変化、短いエフェクト、SE、数値ポップアップなどを活用する
- 緊張と緩和を入れる
  - 危険な局面、回復や強化のタイミング、ボーナスチャンスを交互に配置する
  - 30秒から1分のプレイでも展開が変化するようにする
- 上達できる余地を残す
  - ランダム性だけで結果が決まらないようにする
  - 操作精度、判断速度、ルート選択、リソース管理など、プレイヤーの腕前が出る要素を入れる

**プロトタイプに含めるべき要素**

- 明確な目的
  - 例: 60秒生き残る、スコア100を目指す、ボスを倒す、出口に到達する
- 失敗条件
  - HPが0、時間切れ、敵に捕まる、リソースが尽きるなど
- スコアまたは進行度
  - 前回より上手くなったか比較できる指標を入れる
  - 例: スコア、到達階層、生存時間、コンボ数
- リスクとリターン
  - 危険地帯に行くと高得点、強い攻撃は隙が大きい、アイテムを取るには敵に近づく必要がある、など判断が生まれる構造を入れる
- 段階的な難度上昇
  - 時間経過で敵が増える、ステージが狭くなる、敵の種類が増えるなど
  - 急に理不尽にせず、プレイヤーが変化を理解できるようにする
- 気持ちいい瞬間
  - コンボ、連鎖、クリティカル、回避直後の反撃、まとめて倒す、強化で一気に形勢逆転する場面などを最低1つ入れる
- 速いリスタート
  - ゲームオーバー後にすぐ再挑戦できるようにする
  - タイトル画面や凝ったメニューより、再プレイのテンポを優先する

**作り始める前に決めること**

まず「このゲームで一番面白い瞬間は何か」を1文で決める。

例:

- 敵をギリギリまで引きつけて一網打尽にするのが面白い
- 限られた手札から最善の合成を選ぶのが面白い
- 危険なルートほど報酬が高く、欲張るか迷うのが面白い
- 操作ミスしそうな速度の中で、リズムよく回避するのが面白い

プロトタイプでは、その1文を支える状況、ルール、成功/失敗のフィードバック、再挑戦したくなる指標を優先して実装する。
長いストーリー、多数のステージ、大量のアイテム、凝ったメニュー、複雑な成長システムは、遊びの核が確認できてから追加する。

## 開発環境とコマンド

### 基本コマンド

uvを利用します

```bash
# 実行
uv run python main.py

# pytest実行
uv run pytest tests/
uv run pytest tests/test_module.py

# Ruff実行
uv run ruff check
uv run ruff check <file_name> --fix

# 型チェック(ty)
uv run ty check
uv run ty check <file_name>

# app/webビルド、docs/への配置
uv run pyxel package prototypes/001_alchemy_deckbuilder prototypes/001_alchemy_deckbuilder/main.py
uv run pyxel app2html 001_alchemy_deckbuilder.pyxapp
mv 001_alchemy_deckbuilder.html docs/
```

### Pyxel Tips

Pyxel利用時に気をつけるべきことを完結に記載する

- フォントの制約のため、描画するテキストは半角英数字と一部の記号に限定される
- 日本語や絵文字は描画できないため、ゲーム内のテキストは英語で統一する
